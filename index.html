A Very Fishy Responsive Redesign


Let's Give a Little Context...


In this project I will be walking you through my process of redesigning an antiquated fish auction website into a modern, responsive interface. In it's current state, the Aquabid website has many usability problems. Through a process of both low and high fidelity protyping, and then translating those prototypes into a webapp, I will develop an end product that is more functional than the origial Aquabid interface, and usable across all devices: desktop, mobile, and tablet.

Part 1: Identifying Usability Problems


Picking a Web page


Aquabid.com is one of the most popular websites on the internet for aquarium hobbyists to buy and sell their fish. It is essentially the Ebay of fish. However, I believe it is hindered by its antiquated and difficult to use interface, especially on the "view listings" pages. As someone who is an enthusiastic aquarium hobbyist, I chose this website to bring awareness to how a refresh of the interface could bring about a better experience for both new and experienced users who use Aquabid.

Finding Problems


Problem 1:


Problem:

Large ad space restricts accessibility/visibility of the navigation bar and search bar at top of page.


Usability Issues:

This problem poses issues for both learnability and efficiency. Due to the large ad space it is difficult for both first time users to find where the navbar is located, and reduces the ease by which frequent users could access the navbar.



Problem 2:


Problem:

The nav and search bar is very small.


Usability Issues:

This poses issues for both learnability and efficiency again as first time users will struggle to even find the nav and search bar, and frequent users would have to take additional time to be precise with their clicks to access the tiny bars.



Problem 3:


Problem:

There are many key functionalities that are hidden away in the top bar. For example, the account based functionalities are hidden away in this small, arbitrary bar.


Usability Issues:

This poses issues for learnability and memorability, as it is difficult for first time users to find and it would take a while to find for returning users who haven’t used the interface in a while.



Problem 4:



Problem:

There is a “Show” section that acts as a filter. However it is not very clear on what is being presented, i.e. what is the active filter.


Usability Issues:

This poses issues for learnability and memorability, as it is difficult for first time users to figure out what exactly is being filtered. Same issue could apply with regards to memorability.



Problem 5:



Problem:

In general, the table rows are very tightly stacked on top of eachother, which can make for difficult reading and navigation.


Usability Issues:

This poses issues for efficiency as users of all types need to be very precise in their clicks in order to make sure they are accessing the right element.



Problem 6:



Problem:

There is an incomplete (non-exhaustive) list of icons above the table that indicates certain key attributes of an item.


Usability Issues:

In general, icons like these pose issues for learnability and memorability as first time users will be overwhelmed by the amount of icons and what each of them mean. Infrequent returning users will also forget what each icon means leading to bad memorability.



Problem 7:



Problem:

There is an icon that indicates that a photo is available for the listed fish. However, you need to click onto another page in order to see that photo. It would be better just to have a thumbnail available with the listing on the table.


Usability Issues:

This problem presents issues for learnability, efficiency, and memorability. Yes it is helpful to know that it has a photo, but a user needs to click into a different interface in order to access that photo. Wouldn’t it be better just to have the photo alongside the other listings as a thumbnail to enable comparative shopping?



Problem 8:



Problem:

The column which indicates the amount of time remaining in an auction does not clearly differentiate between those with minutes left vs hours left.


Usability Issues:

This problem presents issues for efficiency as it makes it difficult for all users to compare the time units of each “time remaining” field. It would be better to replace with a format like Replace with consistent format like: 00h 30m 33s.



Problem 9:



Problem:

The bids column does not actually indicate how many bids are listed for a certain fish. It just has this “buynow” logo that you cannot press on.


Usability Issues:

This presents an issue for efficiency. The buy now icon that is listed provides no other information other than that there is a buy now option available. Wouldn’t it be better suited listing the buy now value? Or even the number of bids on the fish?



Problem 10:



Problem:

There is a bottom bar with important information like FAQs etc that is hidden away at the bottom.


Usability Issues:

This poses an issue for learnability and memorability, as these features are tucked away at the very bottom of the page. Posing an issue for the accessibility for first time and infrequent users.



Problem 11:



Problem:

Lack of header hierarchy


Usability Issues:

The lack of header hierarchy creates issues for learnability and memorability as it creates a confusing page structure for new and infrequent users to navigate.



Accessibility


WebAIM WAVE detected numerous accessibility problems with Aquabid.com. The vast majority of these critiques I agreed with, but I do not think it went far enough in its critique as I believe it missed many ways in which the current Aquabid website lacks accessibility. I will begin below by detailing a few key critiques WebAIM WAVE had to offer and whether I agree with those critiques. I will round out by discussing what I think WebAIM WAVE missed.


What WebAIM Suggested


Accessibility Issue 1:


WebAIM WAVE Suggestion

Missing Form Label: A form control does not have a corresponding label.


My Critique

I agree with this assessment. Having just an empty field can be confusing for users and provides no guidance to users who are using audible labeling to navigate the website.



Accessibility Issue 2:


WebAIM WAVE Suggestion

Very low contrast: Very low contrast between text and background colors


My Critique

This one I disagree with. The bold red color contrasts fairly well in my opinion against the white and gray background.



Accessibility Issue 3:


WebAIM WAVE Suggestion

Device dependent event handler: An event handler is present that may not be accessible


My Critique

I agree with this critique. In order to have a table item change color, it must be moused over. Would be better replaced with device agnostic event handler like hover.



What I believe WebAIM missed



Problem Missed

Poor spacing between rows in the table. Most items in the interface are relatively tiny to the frame within which you view the page. This would lead to countless issues in accessibility for those who are visually impaired, have lack of fine motor functions and beyond.
